# План: «Пенальти ЧМ-2026» — браузерная игра по мотивам бота

## Концепция

Отдельный сайт с 2D-аркадой: игрок выбирает сборную ЧМ-2026, бьёт серию пенальти
против ИИ-вратаря и попадает в общую таблицу рекордов. Сборные и их данные
переиспользуются из БД бота (таблица `teams`: 48 команд, группы, рейтинг FIFA).

## Решения (зафиксированы с пользователем 2026-06-10)

| Вопрос      | Решение                                  |
|-------------|------------------------------------------|
| Платформа   | Отдельный сайт (не Telegram Mini App)    |
| Графика     | 2D-аркада                                |
| Персонажи   | Сборные ЧМ-2026 (флаг + цвета формы)     |
| Режим       | Соло против ИИ-вратаря + лидерборд       |

## Стек

| Компонент   | Технология                                   | Почему |
|-------------|----------------------------------------------|--------|
| Игра        | Phaser 3 + TypeScript + Vite                 | стандарт для 2D-веб-игр, отлично работает на мобильных |
| API         | FastAPI (Python 3.11)                        | тот же язык, что и бот; переиспользуем модели/сессию SQLAlchemy |
| БД          | существующий PostgreSQL (новые таблицы)      | сборные уже там; лидерборд — пара таблиц рядом |
| Деплой      | docker-compose: +2 сервиса (api, nginx)      | та же схема, что у бота |

Расположение: монорепо — новая папка `game/` в этом репозитории
(`game/frontend/` — Phaser, `game/api/` — FastAPI). Один compose, одна БД,
один VPS.

## Геймплей (MVP)

- **Удар**: тап/клик задаёт точку прицеливания в воротах, удержание — шкала силы.
  Сильнее удар → быстрее мяч, но больше разброс (может уйти мимо/в штангу).
- **Вратарь (ИИ)**: перед ударом «решает», куда прыгнуть, с вероятностями,
  зависящими от сложности; угадал зону и успел по таймингу → сейв.
- **Серия**: 10 ударов. Гол = очки; подряд забитые — множитель комбо;
  гол в «девятку» — бонус. С каждым ударом вратарь умнее/быстрее.
- **Сборная**: выбор из 48 команд (флаг, цвета футболки бьющего). Рейтинг FIFA
  соперника слегка влияет на силу вратаря — выбор «за кого играть» имеет вкус,
  но не ломает честность лидерборда.
- **Лидерборд**: топ-50 за всё время + топ за неделю. Игрок задаёт ник
  (3–20 символов), привязка по cookie/localStorage-токену — без регистрации.

## Что НЕ входит в MVP

PvP, звук, турнирный режим «пройди весь ЧМ», вход через Telegram,
кастомизация персонажа. Всё это — кандидаты на v2.

## API (черновик)

```
GET  /api/teams                 # 48 сборных: id, name, group, fifa_rank
GET  /api/leaderboard?period=all|week
POST /api/games/start           # создаёт игру на сервере, выдаёт game_token
POST /api/games/{id}/kick       # параметры удара; сервер сам считает гол/сейв
POST /api/games/{id}/finish     # фиксирует результат в лидерборд
```

Ключевое анти-чит-решение: исход каждого удара считает **сервер** (фронт шлёт
точку прицела и силу, сервер бросает кости вратаря и возвращает результат).
Очки в лидерборде нельзя подделать, просто отправив POST с миллионом очков.

## Новые таблицы БД

```
game_players(id, nickname unique, token_hash, created_at)
game_sessions(id, player_id, team_id, started_at, finished_at,
              score, goals, kicks, state jsonb)
```

Миграции — тем же Alembic, что и у бота.

## Фазы и задачи

### Фаза 1 — скелет (≈полдня)

- [ ] `game/frontend`: Vite + TypeScript + Phaser 3, одна сцена `PitchScene`
      (фон-поле, ворота, мяч на точке) — просто рендерится в браузере.
- [ ] `game/api`: FastAPI-приложение, `GET /api/health`,
      `GET /api/teams` — читает 48 сборных из существующей БД
      (переиспользуем `src/models/reference.py` и async-сессию).
- [ ] docker-compose: сервисы `game-api` и `nginx` (раздаёт собранный фронт,
      проксирует `/api` → game-api) с mem_limit/cpus.
- [ ] dev-режим без Docker: `npm run dev` (Vite-proxy на API) + `uvicorn`.
- [ ] Критерий готовности: открываю localhost — вижу ворота; `/api/teams`
      отдаёт реальные сборные.

### Фаза 2 — ядро геймплея (1–2 дня)

- [ ] Прицел: тап/клик по воротам ставит точку удара.
- [ ] Сила: удержание заряжает шкалу; сила = скорость мяча + разброс
      (риск промаха/штанги).
- [ ] Полёт мяча: твин с лёгкой дугой, попадание в зону ворот 5×3.
- [ ] Вратарь: спрайт с прыжком в одну из зон; пока локальный ИИ
      (вероятности по зонам, реакция зависит от номера удара в серии).
- [ ] Исходы: гол / сейв / штанга / мимо + мгновенный фидбек
      (тряска камеры, сетка колышется).
- [ ] Серия из 10 ударов: счёт, комбо-множитель, бонус за «девятку»,
      экран результата.
- [ ] Критерий готовности: в игру интересно сыграть три раза подряд.

### Фаза 3 — сборные и серверный расчёт (≈1 день)

- [ ] Экран выбора сборной: 48 команд по группам, эмодзи-флаг/SVG, цвета формы
      красят спрайт бьющего.
- [ ] `POST /api/games/start` → game_token; состояние серии живёт на сервере.
- [ ] `POST /api/games/{id}/kick`: фронт шлёт (зона, сила, тайминг),
      сервер бросает кости вратаря и возвращает исход — ИИ из Фазы 2
      переезжает на бэкенд один в один.
- [ ] Сила вратаря слегка зависит от fifa_rank соперника.
- [ ] Rate-limit на kick (не чаще ~1 удара в 2 сек на сессию).
- [ ] Критерий готовности: подделать исход удара из консоли браузера нельзя.

### Фаза 4 — лидерборд (≈полдня)

- [ ] Миграции Alembic: `game_players`, `game_sessions`.
- [ ] Ник при первом результате (3–20 символов, фильтр мусора),
      токен в localStorage.
- [ ] `POST /api/games/{id}/finish` — фиксирует счёт (валидируя по состоянию
      сессии на сервере).
- [ ] `GET /api/leaderboard?period=all|week` + экран топ-50 в игре,
      своя позиция подсвечена.
- [ ] Критерий готовности: два игрока с разных устройств видят общий топ.

### Фаза 5 — полировка и деплой (≈1 день)

- [ ] Мобильная адаптация: портретная ориентация, крупные зоны тапа,
      проверка на реальном телефоне.
- [ ] Сочность: частицы при голе, замедление в момент сейва, простые звуки —
      если успеем (звук — кандидат на вылет первым).
- [ ] Экран результата: «поделиться» (ссылка с результатом + позвать в бота).
- [ ] docker-compose.prod: HTTPS (certbot/caddy), домен, healthchecks.
- [ ] Нагрузочная прикидка: лидерборд и kick под параллельными запросами.
- [ ] Критерий готовности: игра открывается по https с телефона «из мира».

## Риски

- **Флаги/наименования**: используем эмодзи-флаги или открытые SVG
  (flag-icons, MIT) — без логотипов FIFA и официальной символики ЧМ.
- **VPS общий**: api+nginx добавить с mem_limit/cpus, как уже сделано
  для бота и БД.
- **Читы**: серверный расчёт закрывает основное; rate-limit на /kick.
