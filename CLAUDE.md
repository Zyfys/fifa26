# CLAUDE.md

Гайд для Claude Code по работе с этим проектом.

## Название и описание

**Прогноз на ЧМ-2026 (Bracket World Cup 2026)** — Telegram-бот «Турнирная сетка ЧМ-2026».

Пользователь проходит весь Чемпионат мира по футболу 2026 и делает собственный прогноз на
результаты всех матчей — от группового этапа до финала. После заполнения всех матчей бот
автоматически рассчитывает таблицы групп, формирует плей-офф и генерирует красивый PDF-файл
с полным прогнозом пользователя.

## Стек технологий

| Компонент    | Технология              |
|--------------|-------------------------|
| Язык         | Python 3.11+            |
| Telegram     | aiogram 3.x             |
| База данных  | PostgreSQL + SQLAlchemy 2.x (async) |
| Миграции     | Alembic                 |
| Конфиг       | pydantic-settings + .env |
| PDF          | (выбрать на этапе планирования: ReportLab / WeasyPrint) |
| Деплой       | Docker + docker-compose (пока локально для теста) |
| Тесты        | pytest + pytest-asyncio |
| Линт/формат  | ruff                    |

## Команды запуска

> ⚠️ Команды-плейсхолдеры. Уточняются на этапе кодинга, после того как появится код.

```bash
# dev — запуск бота локально
python -m src.bot            # (placeholder)

# поднять инфраструктуру (Postgres)
docker-compose up -d

# миграции
alembic upgrade head
alembic revision --autogenerate -m "message"

# tests
pytest

# lint + format
ruff check .
ruff format .
```

## Стиль кода (соглашения)

- Python 3.11+, type hints обязательны.
- Форматирование и линт — `ruff` (line length 100).
- Асинхронность везде где есть I/O (aiogram + async SQLAlchemy).
- Имена: `snake_case` для функций/переменных, `PascalCase` для классов.
- Бизнес-логика — в `services/`, хэндлеры тонкие (только UX + вызов сервисов).
- Никаких секретов в коде — только через `.env` / pydantic-settings.
- Сообщения пользователю — на русском.

## Правила работы

- Сначала план, потом код — никогда наоборот
- Перед большой задачей: декомпозируй на шаги, покажи план
- После каждой фичи: обнови TODO.md
- Если узнал что-то важное: запиши в LESSONS.md
- /compact когда контекст ~50% заполнен
- /clear при переключении на новую несвязанную задачу
- Коммиты после каждой рабочей фичи, не в конце дня
